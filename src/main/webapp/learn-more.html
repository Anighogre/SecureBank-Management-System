How SecureBank Works

User Authentication

Secure login using servlet-based authentication and encrypted session management.

Account Management

View balance, monitor account details, and track transactions in real time.

Fund Transfers

Transfer funds securely with backend validations and controlled workflows.

System Architecture

SecureBank follows a **Java MVC architecture** using Servlets, JSP, and JDBC for backend workflow. The system interacts with an **Oracle Database** for persistent storage of customers, accounts, and transaction records.

  • Client requests handled by Servlet controllers
  • Business logic processed in the Java service layer
  • JDBC interacts with Oracle DB for queries
  • JSP handles the view rendering

Security Highlights

Session Security

Session tokens ensure that only authenticated users access sensitive banking functions.

Data Validation

Every action inside SecureBank is validated to protect user data and prevent manipulation.

Safe Transactions

Funds are transferred only after passing strict backend compliance and security checks.
